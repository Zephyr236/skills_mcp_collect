## 黑盒测试要求

### 1. 测试计划

在执行测试前，先制定测试计划：

```
## {ClassName} 黑盒测试计划

**测试目标：** 验证 {class_name} 外部行为

**正常流程测试用例：**
1. {用例1} - 输入 {输入}，预期 {输出}
2. {用例2} - 输入 {输入}，预期 {输出}

**边界值测试用例：**
1. {用例1} - 输入 {极值}，预期 {结果}
2. {用例2} - 输入 {极值}，预期 {结果}

**错误处理测试用例：**
1. {用例1} - 输入 {错误输入}，预期 {异常/错误响应}
2. {用例2} - 输入 {错误输入}，预期 {异常/错误响应}

**API 契约验证：**
- {契约1}
- {契约2}
```

### 2. 用户确认

**展示测试计划后等待用户确认：**
用户输入「确认」或「ok」后执行测试。

### 3. 执行测试

1. **创建测试文件：** `tests/test_{class_name}_black.py`

2. **编写测试用例**，覆盖：
   - 正常输入输出验证
   - 边界值测试
   - 错误输入处理
   - API 契约验证

3. **使用 pytest + pytest.fixture**：
   - 隔离测试数据
   - 不 mock 自身，只 mock 外部依赖

4. **运行测试：**
   ```bash
   pytest tests/test_{class_name}_black.py -v
   ```

## 失败处理

测试失败时，等待修复后：
1. 重新运行测试
2. 确认全部通过
3. 才算黑盒测试完成

## 重要约束

**仅使用类的公开接口：**
- 不访问私有属性（_开头）
- 不访问内部方法
- 不依赖实现细节

**不 mock 自身：**
- 只 mock 外部依赖
- 自身类的测试使用真实对象
