## 白盒测试要求

### 1. 测试计划

在执行测试前，先制定测试计划：

```
## {ClassName} 白盒测试计划

**测试目标：** 验证 {class_name} 内部实现正确性

**边界条件测试用例：**
1. {用例1} - 预期 {结果}
2. {用例2} - 预期 {结果}

**异常路径测试用例：**
1. {用例1} - 预期抛出 {异常}
2. {用例2} - 预期抛出 {异常}

**Mock 策略：**
- {依赖类A}：使用 unittest.mock.Mock
- {依赖类B}：使用 unittest.mock.patch

**覆盖率目标：** {X}%
```

### 2. 用户确认

**展示测试计划后等待用户确认：**
用户输入「确认」或「ok」后执行测试。

### 3. 执行测试

1. **创建测试文件：** `tests/test_{class_name}_white.py`

2. **编写测试用例**，覆盖：
   - 每个公开方法的边界条件测试
   - 所有异常路径（assertRaises）
   - 内部状态转换验证
   - 依赖 mock（使用 unittest.mock）

3. **运行测试：**
   ```bash
   pytest tests/test_{class_name}_white.py -v
   ```

4. **检查覆盖率：**
   ```bash
   pytest tests/test_{class_name}_white.py --cov={module_name} --cov-report=term
   ```

## 失败处理

测试失败时，等待修复后：
1. 重新运行测试
2. 确认全部通过
3. 才算白盒测试完成
